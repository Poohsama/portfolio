# CSV Upload Scheduler

ECモール向けのCSVファイルを、指定した時刻に自動でアップロードするための業務改善ツールです。

営業時間外や担当者不在時でも更新処理を実行できるようにし、作業待ちや更新漏れによる機会損失を減らすことを目的として開発しました。

## 解決したかった業務課題

ECモールの商品情報や販売設定を更新する際、反映させたい時刻が深夜や早朝になることがあります。

手作業で対応する場合、担当者がその時間まで待機する必要があり、次のような問題がありました。

- 深夜や早朝に手動作業が発生する
- 担当者の負担が大きい
- 対応忘れやアップロードミスが起きる可能性がある
- 更新が遅れることで販売機会を逃す
- 作業できる人が限られ、属人化しやすい

そこで、指定したCSVファイルを決められた時刻に自動でアップロードできる仕組みを作成しました。

## 設計方針

このツールでは、単に自動送信するだけでなく、日常業務で安定して使えることを重視しています。

主な設計方針は次のとおりです。

- 認証情報をソースコードから分離する
- 接続先や対象ファイルを設定ファイルで管理する
- 実行結果をログに残す
- エラー発生時に原因を確認できるようにする
- 入力ファイルが正しい形式か事前に確認する
- 利用者がPythonコードを編集しなくても運用できるようにする

## 主な機能

- 指定時刻でのCSVアップロード
- SFTP接続
- アップロード対象ファイルの存在確認
- CSV見出しの確認
- 実行結果のログ出力
- エラー発生時の通知
- 設定ファイルによる接続先・対象ファイル管理

## 処理の流れ

1. 設定ファイルを読み込む
2. 対象CSVファイルの存在を確認する
3. CSVの見出しを確認する
4. 指定時刻まで待機する
5. SFTPサーバーへ接続する
6. CSVファイルをアップロードする
7. 成功・失敗の結果をログへ記録する
8. 必要に応じて通知を送信する

## 入力ファイルの確認

誤ったCSVをアップロードしないように、処理前に見出しの確認を行います。

たとえば、商品更新用CSVで必要な見出しが存在しない場合は、アップロード処理を中止します。

これにより、別用途のCSVを誤って送信する事故を防ぎます。

## 設定ファイル化

接続先や対象ファイルなど、運用時に変更する可能性がある情報は、ソースコードから分離しています。

```yaml
sftp:
    host: example.com
    port: 22
    username: user

upload:
    local_file: ./input/items.csv
    remote_directory: /upload/
    scheduled_time: "02:00"
```

実際の認証情報や接続先は公開していません。

設定ファイルを変更することで、コード本体を触らずに運用内容を変更できます。

##エラー対策

日常運用で問題が起きた際に原因を追えるよう、次の情報をログへ記録します。

- 実行日時
- 対象ファイル
- 接続結果
- アップロード結果
- エラー内容

エラーが発生した場合でも、どの段階で処理が止まったか確認できるようにしています。

##属人化を減らすための工夫

このツールは、開発者本人だけが使えるスクリプトではなく、ほかの担当者でも運用できることを意識しています。

- 設定変更箇所を限定
- 実行手順を簡略化
- ログによる状態確認
- 入力ファイルの事前検証
- 認証情報と処理コードの分離
- 処理内容が分かるコメントを記載

担当者が変わった場合でも、設定と手順を確認すれば継続して使える構成を目指しています。

##使用技術
- Python
- SFTP
- YAML
- CSV
- ログ出力
- Windows タスクスケジューラ
- Gmail通知
- 実務上の効果

この仕組みにより、担当者が深夜まで待機する必要がなくなりました。

また、営業開始前に更新を完了できるため、更新遅れによる販売機会の損失を減らすことができました。

単なる作業時間の短縮ではなく、業務を人の待機時間から切り離し、安定して実行できる運用へ変更した点がこのツールの目的です。
